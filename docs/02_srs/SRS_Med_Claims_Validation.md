\# Software Requirements Specification (SRS)

\## Medical Claims Validation Portal – Phase 1



\### Purpose

This document describes the requirements for a system that validates medical claims.



\### Scope

The system allows users to:

\- Enter patient details

\- Enter diagnosis and procedure codes

\- Validate claim data

\- Save valid claims



\### Functional Requirements

FR1: User can create a new medical claim  

FR2: System validates mandatory fields  

FR3: System validates diagnosis code format  

FR4: System validates procedure code format  

FR5: System validates claim amount (1–10000)



\### Non-Functional Requirements

\- System should validate data within 2 seconds

\- Error messages should be user-friendly
